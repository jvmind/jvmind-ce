# Security Policy

## Supported Versions

JVMind CE is under active development. Security fixes are applied to the `main` branch and the most recent minor release.

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Send a private report to the maintainers via GitHub Security Advisories:

1. Go to <https://github.com/jvmind/jvmind-ce/security/advisories/new>
2. Describe the vulnerability, reproduction steps, and impact
3. Wait for an acknowledgement before any public disclosure

You should receive an acknowledgement within 7 days. We aim to ship a fix within 30 days, depending on severity and complexity.

## What to Include

- A clear description of the issue
- Steps to reproduce or a proof-of-concept
- The affected versions / commit hashes
- Any known mitigations or workarounds

## Sensitive Data

**Never include** API keys, credentials, full stack traces from production systems, or personally identifying information in your report. We will never ask for your API key.
